# Glossary & NIST 800-53 Pages — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ~1,400 pages to cardinalsixcyber.com: ~1,100 NIST 800-53 Rev 5 control pages and 200-300 glossary term pages, all cross-linked with existing CMMC and 800-171 content, each with practitioner notes.

**Architecture:** JSON data files seeded from NIST OSCAL (800-53) and hand-curated (glossary). New Astro components (`ControlDetail.astro`, `GlossaryEntry.astro`) render detail pages via `getStaticPaths()`. Existing `ComplianceDetail.astro` updated to link 800-53 references instead of showing text badges. Same Docker/nginx deploy.

**Tech Stack:** Astro (static), Python 3 (data seeding), vanilla JS (index filtering), existing nginx/Docker deploy

**Spec:** `docs/superpowers/specs/2026-04-01-glossary-and-800-53-design.md`

### Task 1: NIST 800-53 Data Seeding Script

**Files:**
- Create: `scripts/seed-800-53-data.py`
- Create: `data/nist-800-53.json`

- [ ] **Step 1: Create the seed script**

Write `scripts/seed-800-53-data.py` that:
1. Fetches the NIST OSCAL 800-53 Rev 5 catalog from `https://raw.githubusercontent.com/usnistgov/oscal-content/main/nist.gov/SP800-53/rev5/json/NIST_SP-800-53_rev5_catalog.json`
2. Parses each control and control enhancement from the OSCAL catalog
3. Extracts: id, title, family, description, supplemental guidance, related controls
4. Slug generation: `AC-2` → `ac-2`, `AC-2(1)` → `ac-2-1` (replace parens with hyphen, strip trailing paren, lowercase)
5. Cross-references with existing `data/cmmc-practices.json` and `data/nist-800-171.json`: scans their `nist80053` fields to build reverse mappings (`cmmcPractices` and `nist800171` arrays on each 800-53 control)
6. Outputs `data/nist-800-53.json` matching the spec schema
7. Uses `os.makedirs('data', exist_ok=True)` and `encoding='utf-8'` on all file ops

The 20 800-53 Rev 5 families: AC, AT, AU, CA, CM, CP, IA, IR, MA, MP, PE, PL, PM, PS, PT, RA, SA, SC, SI, SR.

- [ ] **Step 2: Run the seed script**

```bash

```
Verify:

Expected: ~1,000+ controls across 20 families.

### Task 2: Glossary Data Seeding Script

**Files:**
- Create: `scripts/seed-glossary-data.py`
- Create: `data/glossary.json`

- [ ] **Step 1: Create the seed script**

Write `scripts/seed-glossary-data.py` that:
1. Contains a comprehensive list of 200-300 cybersecurity terms with definitions and "why it matters" text — all written for non-technical business owners
2. Terms cover: CMMC terminology (CMMC, CUI, FCI, SSP, POAM, C3PAO, etc.), RMF process (ATO, IATT, RMF, ISSM, ISSO, etc.), NIST frameworks (800-53, 800-171, OSCAL, etc.), DoD terms (CORA, STIG, ACAS, ESS, DISA, etc.), general cybersecurity (MFA, zero trust, phishing, ransomware, encryption, firewall, VPN, SOC, SIEM, etc.)
3. Generates `relatedResources` for each term by scanning `data/cmmc-practices.json`, `data/nist-800-171.json`, and `data/nist-800-53.json` — matches term name (case-insensitive) against titles and domain/family names, max 8 per term
4. Slug generation: term name → lowercase, spaces to hyphens, strip special chars (e.g., "Authority to Operate (ATO)" → `authority-to-operate-ato`)
5. Outputs `data/glossary.json` matching the spec schema

This is a large script since it contains all 200-300 term definitions inline. The definitions are the content — no external data source.

**Note:** The glossary seed script produces the final content (definition + whyItMatters) at generation time, satisfying the spec's practitioner notes requirement for glossary terms. Unlike 800-53 controls (which get notes injected in a separate Task 10), glossary content is written directly into the seed script since there's no external data source to merge with. No separate glossary notes injection task is needed.

- [ ] **Step 2: Run the seed script**

```bash

```
Verify:
```bash

```
Expected: 200-300 terms, majority with related resources.
